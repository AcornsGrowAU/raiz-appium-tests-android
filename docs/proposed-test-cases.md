# Proposed Test-Case Backlog (Ranked)

**Summary.** 45 cases were proposed across the fan-out; after adversarial cross-check **38 are KEPT** (verdict keep/refine) and **7 are CUT** (already-covered, infeasible-as-written, or pure compositions of harder cases). Of the kept, **29 are non-deferred** (build-now candidates) and **9 are DEFERRED** (rewards / super / auth / milestones / plans-fees — await user context). Top themes: **cross-feature money-conservation invariants** (transfer/routing/no-cross-post — the single biggest novel value gap), **value-not-presence upgrades** (read the real number, not just that a `$` rendered), **negative / enforcement-point oracles** (below-minimum, over-balance, tier/entitlement gating — asserted at COMMIT, not the keypad), and **per-entity independence** (per-kid / per-jar portfolio + recurring storage).

Ranking key: (a) cross-check verdict (cut dropped), (b) confidence desc, (c) consensus_count desc, (d) priority. `keep` ranks above `refine` at equal confidence.

## Highest-value-first shortlist — top 12 to build next

Ordered to front-load the **cross-feature conservation / value oracles** (the suite's biggest documented gap: money-conservation discipline) and the highest-confidence, genuinely-novel value upgrades. Notes call out the cross-checkers' refinements.

1. **new-kid-zero-start** (P0, conf 82, cons 20) — bare unfunded kid renders exactly $0.00, not blank/null/inherited. Strongest of the batch. *Author a BARE kid_user variant (existing helper hardcodes 'funded'+Aggressive); confirm the bare kid renders on the parent list rather than an 'Add a kid' empty card. API current_balance==0.0 is the reliable anchor.*
2. **jar-six-cap-enforced** (P1, conf 86, cons 15) — 7th jar blocked, six unchanged. *Assert NOT-completable + count==6 (don't hard-code the at-limit affordance/string); seed bare jars to skip 6× ACH settle. Build the shared "seed N sub-accounts, attempt N+1, assert blocked + count stable" helper here — kid-eight-cap reuses it.*
3. **deposit-main-routing-isolation** (P0, conf 80) — $10 into Main, every jar/kid delta==0. *delta==0 is the load-bearing assertion; scope the Main-rise leg soft (async settle → pending/band, not exact +10).*
4. **home-total-conservation** (P1, conf 80) — Home total == Main + Σ jar cards + Σ kid cards + backend cross-check. *Use DELTA_BAND (cards are market-priced); avoid Moderate (prices to $0). De-dupe vs #6 by scoping this to the HOME headline + per-card sum.*
5. **jar-target-roundtrip** (P0, conf 82) — set $5,000, reads back $5,000.00 (catches $5,000→$500 truncation). *UI-set + UI-read-back; downgrade backend goal clause to "if observable". Builds the Manage-Jar detail target reader that jar-goal-progress-ring then extends.*
6. **networth-total-investments-recon** (P1, conf 72) — My Finance 'Total in investments' vs an INDEPENDENT backend oracle (so screen + headline can't share one wrong source). *De-dupe vs #4: scope THIS to the My-Finance screen value vs backend.*
7. **main-jar-transfer-conserves** (P0, conf 72, cons 17) — **API-layer half only**: seeded Owner↔Jar transfer conserves the sum, all other accounts delta==0. *Use small EXACT ACH balances, not the repricing six-figure buffer. On-device transfer half is DEFERRED (needs a JarTransfer page object).*
8. **kid-fund-no-cross-post** (P0, conf 70, cons 17) — **API-layer half**: credit Kid-A +25 → Kid-A delta==+25, Kid-B delta==0.00 exactly. *Implement at API first (no new tooling, no market drift); on-device parent-side lump-sum is a separate deferred unit. Drop the parent-funding-debit clause.*
9. **portfolio-style-allocation-weights** (P0, conf 76, cons 17) — Aggressive ~90/10 vs Conservative inverse, each sums 100±0.5, row-sets NOT identical. *Phase-0: confirm Conservative/Moderate portfolio_name accepted by gen API; use tolerance + differ + sum-to-100 as the robust core, don't hard-code exact published %s.*
10. **per-jar-portfolio-independent** (P2, conf 78) — seeded jar='Conservative' vs Main='Aggressive', A!=B (independent storage). *Scope to the seeded backend read-back (jar's own token); mutate-and-reread is a stretch needing a Manage-Jar portfolio page object.*
11. **per-kid-portfolio-independent — static half** (P1, conf 60→higher once split, cons 15) — kid-A==Aggressive, kid-B==Moderate, A!=B (assert LABEL not balance; Moderate prices to $0). *Requires extending kid_user with a portfolio_name arg (currently shared constant). On-device change-no-bleed deferred.*
12. **inflow-triple-oracle** (P1, conf 70) — seeded $X lump-sum ties together a typed History Buy row + value-tile (== seeded current_balance, absolute not delta) + a typed backend Investment row. *The txn-History typed-row leg is the strongest novel piece; drop the literal "+$X delta" (gen users have no price history).*

**Build-order dependencies to honor:** ship the enforced-cap helper at #2 before kid-eight-cap; ship the Manage-Jar detail reader at #5 before jar-goal-progress-ring; the four conservation cases (#3, #7, #8 + withdraw-over-balance) all want **small exact ACH balances**, never the repricing rich buffer. Tier-gating-kids-jars, withdraw-over-balance, deposit-sub5 and pending-vs-settled are all **spike-gated** (verdict refine, confidence ≤63) — run their Phase-0 probes before committing them to a sprint.
